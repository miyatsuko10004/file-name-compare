# ファイル名比較ツール

このツールは、OCR処理などの前後でファイルが欠落していないかを確認するために、2つのディレクトリ間でファイル名を比較するPythonスクリプトです。拡張子を無視してファイル名を比較し、一方のディレクトリには存在するがもう一方には存在しないファイルを特定します。

## 概要

PDFファイルからMarkdownファイルへのOCR変換処理などにおいて、正しく変換されなかったり、ファイルが欠落したりするケースがあります。このツールは、変換元ディレクトリと変換先ディレクトリを比較し、どのファイルが欠落しているかを効率的に特定するのに役立ちます。

## インストール

1.  Pythonがインストールされていることを確認してください。
2.  `uv` を使用して依存関係を管理します。まだ `uv` がインストールされていない場合は、以下のコマンドでインストールできます。
    ```bash
    pip install uv
    ```
3.  プロジェクトのディレクトリに移動し、`uv` を初期化します。
    ```bash
    cd your-project-directory
    uv init
    ```
4.  （もし `uv.lock` に依存関係が記述されていれば、`uv install` を実行します）

## 使用方法

以下のコマンドを実行して、2つのディレクトリを比較します。

```bash
uv run check_missing_files.py <比較元ディレクトリのパス> <比較先ディレクトリのパス>
```

**例:**

例えば、`./pdfs` ディレクトリに元のPDFファイルがあり、`./markdowns` ディレクトリにOCR変換後のMarkdownファイルがあるとします。この場合、OCR変換後に欠落しているMarkdownファイル（元のPDFは存在するが、対応するMarkdownファイルがないもの）を特定するには、次のように実行します。

```bash
uv run check_missing_files.py ./pdfs ./markdowns
```

### 引数

*   `<比較元ディレクトリのパス>`: 比較の基準となるディレクトリのパスです。このディレクトリ内のファイルが、比較先ディレクトリに存在するかどうかがチェックされます。（例: 元のPDFファイルがあるディレクトリ）
*   `<比較先ディレクトリのパス>`: 比較元ディレクトリ内のファイルに対応するファイルが存在するかどうかを確認するディレクトリのパスです。（例: OCR変換後のMarkdownファイルがあるディレクトリ）

### 出力

ツールは、比較元ディレクトリには存在するものの、比較先ディレクトリには対応するファイルが見つからなかったファイル名（拡張子なし）のリストを標準出力に表示します。
すべてのファイルが対応している場合は、その旨がメッセージとして表示されます。
